# Milestone v1.0: Simulation-to-Inference Pipeline

**Status:** SHIPPED 2026-04-07
**Phases:** 1-7
**Total Plans:** 18

## Overview

Build a validated HGF-based analysis pipeline for the PRL pick_best_cue task. The project progresses from task environment reproduction through model definition, simulation, fitting, recovery validation, group-level hypothesis testing, and an interactive parameter exploration GUI. Each phase produces independently verifiable outputs.

## Milestone Summary

**Key Decisions:**

- pyhgf 0.2.8 chosen as HGF library (JAX-backed, PyMC integration, Network API)
- 3 parallel binary HGF branches with shared volatility parent (literature default)
- Softmax + stickiness response model (beta, zeta)
- Partial feedback: only chosen cue receives reward input
- Random-effects BMS (Rigoux et al. 2014) for model comparison
- ds_env (Python 3.10 conda env) as installation environment
- Two-Op split pattern (_GradOp + _LogpOp) wrapping JAX lax.scan for PyMC
- Batch forward pass for GUI slider updates (~1.1-1.7s vs ~4.2s trial-by-trial)
- bambi 0.15.0 for Python 3.10 compatibility
- groupBMC 1.0 for Rigoux et al. 2014 VB algorithm

**Issues Resolved:**
- pyhgf observed mask requires int dtype (not bool) for JAX tracing
- 3-level model NaN boundary at omega_2 >= ~-1.2 (handled by prior upper=0.0)
- conda run multi-line script failure on Windows (workaround: temp script file)
- Ruff I001 unsorted imports in GUI module

**Issues Deferred:**
- omega_3 parameter recovery known to be poor (literature caveat)
- System Python 3.13 incompatible with pyhgf 0.2.8
- Full 180-participant batch ~3-4.5 hours sequential on CPU
- WAIC batch runtime budget 30-60 min
- mason2024 BibTeX page details need verification

**Technical Debt Incurred:**
- cores=1 default on Windows; re-test cores=4 if batch runtime exceeds 8 hours
- Quarto cache disabled during development (switch to true after results stabilise)
- 07-02 notebook verified working by user but no formal SUMMARY.md written
