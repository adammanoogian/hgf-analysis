Create `src/prl_hgf/models/response.py` with:

- `from __future__ import annotations`
- Imports: `jax.numpy as jnp`, `from prl_hgf.models.hgf_2level import BELIEF_NODES` (for node index constants)
- Function `softmax_stickiness_surprise(hgf, response_function_inputs, response_function_parameters) -> float`:
  - This signature is MANDATORY for compatibility with `pyhgf Network.surprise()` — do not change it
  - `response_function_inputs`: choices array, shape `(n_trials,)`, values 0/1/2
  - `response_function_parameters`: JAX array `[beta, zeta]` — `beta = params[0]`, `zeta = params[1]`
  - Extract `expected_mean` from continuous-state BELIEF_NODES (1, 3, 5) using `hgf.node_trajectories[idx]["expected_mean"]` — these are the log-odds beliefs (NOT the binary node expected_mean which is sigmoid-transformed)

  CRITICAL DECISION on which `expected_mean` to use:
  - The RESEARCH.md Pattern 6 code sample uses `hgf.node_trajectories[1]["expected_mean"]` (continuous-state node) for mu1 in the softmax
  - The RSP-02 requirement says: "mu_1k is the sigmoid-transformed level-1 belief for cue k"
  - Resolution: Use `hgf.node_trajectories[binary_idx]["expected_mean"]` from BINARY nodes (0, 2, 4) which IS the sigmoid-transformed probability P(reward|cue k). The softmax formula P(choice=k) = exp(beta * P_reward_k + zeta * sticky_k) makes more sense with probabilities in [0,1] than with unbounded log-odds. This matches RSP-02 exactly.
  - So: `mu1_0 = hgf.node_trajectories[0]["expected_mean"]` (binary node 0), `mu1_1 = hgf.node_trajectories[2]["expected_mean"]` (binary node 2), `mu1_2 = hgf.node_trajectories[4]["expected_mean"]` (binary node 4)
  - Define `INPUT_NODES_FOR_RESPONSE = (0, 2, 4)` constant

  - Stack into shape `(n_trials, 3)`: `mu1 = jnp.stack([mu1_0, mu1_1, mu1_2], axis=1)`
  - Build stickiness indicator: `prev_choices = jnp.concatenate([jnp.array([-1]), choices[:-1]])` — sentinel -1 for first trial (RSP-04)
  - Stickiness matrix shape `(n_trials, 3)`: for each cue k, `(prev_choices == k).astype(float)`
  - Compute logits: `logits = beta * mu1 + zeta * stick`
  - Log-softmax (numerically stable): `log_probs = logits - jnp.log(jnp.sum(jnp.exp(logits), axis=1, keepdims=True))`. Alternatively use `jax.nn.log_softmax(logits, axis=1)` if available.
  - Index chosen cue log-prob: `trial_loglik = log_probs[jnp.arange(n_trials), choices.astype(int)]`
  - Total surprise (negative log-likelihood): `surprise = -jnp.sum(trial_loglik)`
  - Guard against NaN: `return jnp.where(jnp.isnan(surprise), jnp.inf, surprise)`
  - NumPy-style docstring explaining parameters, return value, and the formula

Update `src/prl_hgf/models/__init__.py`:
- Add import of `softmax_stickiness_surprise` from `.response`
- Add to `__all__` if using explicit exports

IMPORTANT: The response function must work with JAX arrays throughout (not numpy). The `choices` array passed as `response_function_inputs` may be a numpy array from the caller — cast to jnp inside the function if needed. Use `jax.nn.log_softmax` if it exists in the installed JAX version for cleaner code, otherwise use the manual formula.

- `test_zero_zeta_no_stickiness_effect`: With zeta=0.0, surprise should be identical regardless of previous choice pattern (same current beliefs, same current choice). Test by comparing two sequences that differ only in previous choices.
