Create `src/prl_hgf/models/hgf_2level.py` with:

- `from __future__ import annotations` at top
- Import `Network` from `pyhgf.model`
- A module-level constant `N_CUES = 3`
- Node index constants: `INPUT_NODES = (0, 2, 4)`, `BELIEF_NODES = (1, 3, 5)` — export these for downstream use
- Factory function `build_2level_network(omega_2: float = -4.0) -> Network`:
  - Creates a `Network()` instance
  - Adds 3 pairs of (binary-state, continuous-state) nodes using `add_nodes()` in a loop over cues
  - Each binary-state node is created with `add_nodes(kind="binary-state")`
  - Each continuous-state node is created with `add_nodes(kind="continuous-state", value_children=binary_idx, node_parameters={"tonic_volatility": omega_2})`
  - Sets `net.input_idxs = INPUT_NODES`
  - Returns the network
- A helper function `prepare_input_data(trials: list, choices: list[int], rewards: list[int]) -> tuple[np.ndarray, np.ndarray]`:
  - Takes trial list, choice indices (0/1/2), and reward outcomes (0/1)
  - Returns `(input_data, observed)` arrays both shape `(n_trials, 3)` with int dtype for observed
  - For each trial t: `input_data[t, choices[t]] = rewards[t]`, `observed[t, choices[t]] = 1`
  - All other entries stay 0
- A helper function `extract_beliefs(net: Network) -> dict[str, np.ndarray]`:
  - Extracts from `node_trajectories` after forward pass
  - Returns dict with keys: `"mu1_cue0"`, `"mu1_cue1"`, `"mu1_cue2"` (from continuous nodes 1, 3, 5 — use `"expected_mean"` which is log-odds belief), and `"p_reward_cue0"`, `"p_reward_cue1"`, `"p_reward_cue2"` (from binary nodes 0, 2, 4 — use `"expected_mean"` which is sigmoid-transformed probability)
- NumPy-style docstrings on all public functions and the module

Create `src/prl_hgf/models/hgf_3level.py` with:

- Same imports plus the shared constants from hgf_2level
- Additional constant: `VOLATILITY_NODE = 6`
- Factory function `build_3level_network(omega_2: float = -4.0, omega_3: float = -4.0, kappa: float = 1.0) -> Network`:
  - Same 3-branch construction as 2-level
  - THEN adds the shared volatility parent: `net.add_nodes(kind="continuous-state", volatility_children=([1, 3, 5], [kappa, kappa, kappa]), node_parameters={"tonic_volatility": omega_3})`
  - Sets `net.input_idxs = INPUT_NODES`
  - Returns the network
- Override/extend `extract_beliefs` to also include `"mu2_volatility"` (node 6 `"mean"`) and `"sigma2_volatility"` (1.0 / node 6 `"precision"`) — either by calling the 2-level version and adding keys, or as a standalone function `extract_beliefs_3level(net: Network) -> dict[str, np.ndarray]`
- NumPy-style docstrings

- Brief module docstring describing the two model variants

Use `expected_mean` from binary nodes for P(reward), not `mean`. Follow three-layer naming: omega_2/omega_3/kappa as parameter names at the API boundary, tonic_volatility/volatility_children internally when passed to pyhgf.
