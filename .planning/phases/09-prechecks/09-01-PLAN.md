<objective>
Build the core precheck module (`power/precheck.py`) that gates which HGF parameters
advance to the power analysis sweep. This plan implements PRE-01 (recovery on 50
participants), PRE-02 (confound matrix), PRE-03 (eligibility list with reasons),
and PRE-06 (MCMC convergence gating).

Purpose: Without this gate, the expensive power sweep in Phase 10 could waste
hundreds of cluster-hours on parameters that are not recoverable. The precheck
establishes which parameters are power-eligible before any sweep runs.

Output: `power/precheck.py` with `make_trial_config`, `run_recovery_precheck`,
and `build_eligibility_table`; pipeline script `09_run_prechecks.py`; unit tests.
</objective>

Create `src/prl_hgf/power/precheck.py` with these functions:

1. **`make_trial_config(base: AnalysisConfig, target_total_trials: int) -> AnalysisConfig`**
   - Uses `dataclasses.replace` bottom-up (same pattern as `make_power_config`)
   - Computes scale factor: `(target_per_set - transfer_n_trials) / sum(phase.n_trials)`
   - Scales each `PhaseConfig.n_trials` by the factor with `max(1, round(...))`
   - Transfer n_trials is untouched
   - Logs actual total after construction for verification
   - Do NOT modify `power/config.py` — this is a new function in precheck.py

2. **`run_recovery_precheck(config: AnalysisConfig, n_participants: int = 50, model_name: str = "hgf_3level", seed: int = 42, output_dir: Path | None = None) -> PrecheckResult`**
   - Calls `make_power_config(config, n_per_group=n_participants, effect_size_delta=0.0, master_seed=seed)` to set participant count without any effect size shift
   - Calls `simulate_batch(power_cfg)` to generate data
   - CRITICAL: Filters to baseline only BEFORE fitting: `sim_df_pre = sim_df[sim_df["session"] == "baseline"].copy()` — this avoids 3x compute cost
   - Calls `fit_batch(sim_df_pre, model_name=model_name, cores=1)` with default MCMC settings (4 chains, 1000 draws, 1000 tune)
   - Computes convergence exclusion count: `fit_df.groupby("participant_id")["flagged"].any().sum()` — print count to console
   - Calls `build_recovery_df(sim_df_pre, fit_df, exclude_flagged=True, min_n=30)`
   - Calls `compute_recovery_metrics(recovery_df)` and `compute_correlation_matrix(recovery_df)`
   - Calls existing `plot_recovery_scatter` and `plot_correlation_matrix` for output figures
   - Returns a `PrecheckResult` dataclass containing `metrics_df`, `corr_df`, `n_flagged`, `n_total`, `recovery_df`
   - If output_dir is provided, saves CSVs and PNGs there

3. **`build_eligibility_table(metrics_df: pd.DataFrame) -> pd.DataFrame`**
   - Iterates over `metrics_df` rows
   - For each parameter:
     - If `parameter == "omega_3"`: status = `"exploratory -- upper bound"`, reason includes the actual r value and the project decision rationale
     - If `passes_threshold` is True: status = `"power-eligible"`, reason = `"r={r:.2f} >= 0.7"`
     - If `passes_threshold` is False: status = `"excluded"`, reason = `"r={r:.2f} < 0.7 threshold"`
   - Returns DataFrame with columns: `["parameter", "r", "status", "reason"]`

4. **`PrecheckResult` dataclass** (frozen=True):
   - Fields: `metrics_df`, `corr_df`, `eligibility_df`, `n_flagged`, `n_total`, `recovery_df`

Update `src/prl_hgf/power/__init__.py` to re-export `make_trial_config`, `run_recovery_precheck`, `build_eligibility_table` from precheck.py.

- `from __future__ import annotations` at top
- NumPy-style docstrings for all public functions
- Type hints with Python 3.10+ syntax
- Logging via `logging.getLogger(__name__)`
