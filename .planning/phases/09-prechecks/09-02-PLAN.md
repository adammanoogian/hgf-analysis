<objective>
Add the trial count sweep (PRE-04, PRE-05) and VIZ-01 precheck figure to the
precheck module. The sweep fixes N=30/group and varies total trials across
[150, 200, 250, 300, 420], running recovery at each point with reduced MCMC
settings to identify the minimum trial count where all power-eligible parameters
achieve r >= 0.7.

Purpose: Without knowing the minimum trial count, the power sweep in Phase 10
could run with too few trials (wasting compute on unrecoverable fits) or too
many (unnecessary experiment length for participants).

  <action>
Add three new public functions to the existing `power/precheck.py` (do not modify
the existing functions from Plan 01):

1. **`run_trial_sweep(config: AnalysisConfig, trial_grid: list[int] | None = None, n_per_group: int = 30, model_name: str = "hgf_3level", seed: int = 42, output_dir: Path | None = None) -> list[SweepPoint]`**

   - Default `trial_grid = [150, 200, 250, 300, 420]`
   - For each target_trials in trial_grid:
     a. Call `make_trial_config(config, target_total_trials=target_trials)` to scale trial count
     b. Call `make_power_config(trial_cfg, n_per_group=n_per_group, effect_size_delta=0.0, master_seed=seed + idx)` — use `seed + idx` so each grid point gets a different seed (fresh participants, no between-condition correlation)
     c. Call `simulate_batch(power_cfg)`
     d. Filter to baseline only: `sim_df_pre = sim_df[sim_df["session"] == "baseline"].copy()`
     e. Call `fit_batch(sim_df_pre, model_name=model_name, cores=1, n_chains=2, n_draws=500, n_tune=500)` — REDUCED MCMC settings for the sweep (not inference-grade, but adequate for recovery estimation)
     f. Compute convergence exclusion count per grid point
     g. Call `build_recovery_df(sim_df_pre, fit_df, exclude_flagged=True, min_n=0)` — pass `min_n=0` because small trial counts may lose many participants to convergence failures
     h. Call `compute_recovery_metrics(recovery_df)`
     i. Append `SweepPoint(trial_count=actual_total, metrics_df=metrics_df, n_flagged=..., n_total=...)` to results
     j. Print progress: `"[{idx+1}/{len(trial_grid)}] trials={actual_total}: {n_flagged} flagged, computing recovery..."`
   - If output_dir provided, save `trial_sweep_results.csv` (long-form: trial_count, parameter, r, p, bias, rmse, n, passes_threshold)
   - Return list of SweepPoint

2. **`plot_trial_sweep(sweep_results: list[SweepPoint], r_threshold: float = 0.7, save_path: Path | None = None) -> plt.Figure`**
   - X axis: total trials (from each SweepPoint.trial_count)
   - Y axis: Pearson r
   - One line per parameter, using distinct colors/markers (seaborn color palette or manual)
   - Horizontal dashed red reference line at r=0.7
   - omega_3 line should be dashed/dotted and labeled with "(exploratory)" suffix
   - Legend outside or inside plot, clearly identifying each parameter
   - Title: "Parameter recovery vs trial count"
   - If save_path provided, save as PNG at 150 dpi
   - Use matplotlib Agg backend (same as existing plots.py)

3. **`find_minimum_trial_count(sweep_results: list[SweepPoint], r_threshold: float = 0.7) -> int | None`**
   - Examines sweep_results in ascending trial_count order
   - Returns the smallest trial_count where ALL power-eligible parameters (those with status != "excluded" in the eligibility table — i.e., parameters that pass r >= 0.7 at the baseline 420 trial count) have r >= r_threshold
   - Excludes omega_3 from the "all must pass" requirement (it is exploratory)
   - Returns None if no trial count satisfies the condition
   - Note: This needs to know which parameters are power-eligible. Accept an optional `eligible_params: list[str] | None` argument; if None, use all parameters except omega_3.

4. **`SweepPoint` dataclass** (frozen=True):
   - Fields: `trial_count: int`, `metrics_df: pd.DataFrame`, `n_flagged: int`, `n_total: int`

Also update `__all__` in precheck.py to include the new exports. Do NOT update `power/__init__.py` re-exports in this task (keep it simple — the new functions are importable via `from prl_hgf.power.precheck import ...`).

Follow existing conventions: `from __future__ import annotations`, NumPy-style docstrings, type hints with 3.10+ syntax, ruff-clean.
