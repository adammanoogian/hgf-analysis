<objective>
Produce the Phase 21 graph scope memo `benchmark-scope-memo.md`: quantify how big the
batched hierarchical PAT-RL logp graph actually is (HLO op count, scan-body depth,
FLOPs/draw) and contextualize its size against state-space-model baselines (NumPyro
HMM, Kalman/LGSSM). This is Wave 1 local desk work; no cluster, no GPU, no new
production code.

**In scope:**
- Run `jax.make_jaxpr` and `jax.xla_computation` on `_build_patrl_log_posterior` (3-level
  PAT-RL) and `build_logp_fn_batched` (pick_best_cue) locally on CPU at small tractable
  P (P=3 is the CONTEXT.md-decided tractable size); count HLO ops in the compiled graph
  and in the per-trial scan body separately.
- Extract the scan-body-depth column empirically (count the ops XLA sees inside a single
  `beliefs_propagation` invocation, including the nested `@partial(jit, ...)` sub-functions
  and the per-trial `lax.cond`).
- Derive FLOPs/draw by multiplying the compiled HLO op count by leapfrog-steps-per-draw
  (typical NUTS: ~16-32 steps, 2 grad evaluations per step); state assumptions explicitly.
- Populate a markdown comparison table against state-space baselines (NumPyro HMM
  100-state, Kalman filter LGSSM). RESEARCH.md already provides the per-step op estimates
  for these baselines — cite them.
- Document the structural differences the HLO op counts expose: pyhgf's nested JIT
  decorators inside the scan body, the Layer-2 `tree_map + jnp.where` clamp traversing
  the full attributes pytree per trial, the closure-over-data pattern in
  `build_logp_fn_batched_patrl:507-512`.

**Explicitly NOT in scope:**
- Naming a verdict on the 2197s cold / 1965s warm observation (verdict belongs to 21-07).
- Running anything on GPU (this plan is CPU-only desk work).
- Any code change to production modules (the memo is the only artifact; HLO-dumping
  helper code may live in a new diagnostic module but is not required if the
  `jax.make_jaxpr` output can be generated inline in a notebook or ad-hoc Python REPL).
- Comparing against Bayesian workhorses (8-schools, Radon) or ML forward passes
  (transformer, CNN, BNN) — CONTEXT.md explicitly excludes these.
- Measuring wall-clock JIT time (that's Wave 2's job; Wave 1 measures graph size, not
  compilation time).

Purpose: Give VERDICT.md (21-07) an empirical "is this graph structurally huge or
small?" answer. If the PAT-RL scan body has ~100-140 HLO ops/trial and NumPyro HMM
has ~20-30, the graph IS large; that shapes the verdict toward a graph-level fix. If
the scan-body depth is comparable to baselines, the verdict shifts toward a
sampler/cache-key-level fix.

Output:
- `.planning/phases/21-benchmark-bottleneck-diagnosis/benchmark-scope-memo.md` with
  the required columns (scan-body depth, HLO op count, FLOPs/draw), state-space
  comparison table, and structural-difference narrative.
- `.planning/phases/21-benchmark-bottleneck-diagnosis/21-01-SUMMARY.md` recording
  headline numbers and any anomalies discovered during the jaxpr trace.
