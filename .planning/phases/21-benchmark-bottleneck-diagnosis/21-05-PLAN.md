<objective>
Produce `jit_scaling_sweep.csv` with authoritative cross-process persistent-cache
measurements for P ∈ {5, 20, 50} — the P-sweep deliverable from Phase 21 SC4.
The distinction matters: 21-04's NUTS probe wrote `same_proc_warm_s` (second call
within one Python process → tests in-process tracing cache). This plan writes
`next_proc_warm_s` (second SLURM job → tests persistent compilation cache). If
`next_proc_warm_s > same_proc_warm_s * 2`, the persistent cache is not landing —
a direct SC4 verdict signal.

**In scope:**
- A parametrized SLURM script `cluster/21_diagnose_pscan.slurm` that accepts
  `--export=ALL,PROBE_P=N,PROBE_ROLE=cold|warm` and runs one cold OR one warm probe
  at the specified P.
    - Cold role: invokes `scripts/08_run_power_iteration.py --p-scan N --diagnostic-mode`.
      The probe (from 21-02) runs two in-process calls at N, writing a CSV row with
      `cold_jit_s` + `same_proc_warm_s` filled; `next_proc_warm_s` remains empty.
    - Warm role: invokes the same `--p-scan N --diagnostic-mode` BUT then runs an
      inline python merge block that parses the probe's cold_jit_s field from the
      JAX_LOG_COMPILES log / stdout and writes it to `next_proc_warm_s` of the
      pre-existing row for P=N (the cold row from the prior SLURM job).
    - Each role captures JAX_LOG_COMPILES → `pscan_P{N}_{cold,warm}.log`.
    - Each role dumps XLA text-IR to `hlo_dumps/pscan_P{N}_{cold,warm}/`.
    - Each role git-pushes its slice at exit.
- A submission wrapper `cluster/21_submit_pscan_chain.sh` that orchestrates the
  afterok chain:
    - For P in 5 20 50: submit cold (capture JOBID), submit warm with
      `--dependency=afterok:$JOBID_COLD`.
    - Print all JOBIDs at the end.
    - Walk-away semantics: the wrapper returns immediately after submission; the
      cluster does the sequential work.
- `checkpoint:human-action` task for submission + a follow-up verify task.

**Explicitly NOT in scope:**
- P=2 — CONTEXT.md scaling argument: too small to be informative.
- P=100 — never in requirements; P=50 is the production shape (STATE #105 grid).
- Combining with 21-04's NUTS probe jobs. 21-04 populates the P=5 and P=20 rows
  already; 21-05 ADDS the P=50 row and fills the `next_proc_warm_s` column across
  all three rows by submitting separate cold+warm pairs.
- Running the VB-Laplace P-sweep. 21-03 covered Laplace at P=5,20 (matching
  Laplace-vs-NUTS shapes); adding a Laplace P=50 is a post-Phase-21 follow-up if the
  verdict calls for it.
- Bumping the walltime beyond 30min. If P=50 cold exceeds 30min, the status is
  `infeasible` and that's the deliverable (a P=50 infeasibility signal IS the
  answer).

Purpose:
- Populate the third column (`next_proc_warm_s`) of jit_scaling_sweep.csv at
  P=5, 20, 50 — the cross-process persistent-cache measurement.
- The `cold_jit_s / next_proc_warm_s` ratio at matched P is the authoritative cache
  speedup number. If it's ~1.0 everywhere, the cache is not functioning. If it's
  ~3-10x at P=5 but ~1-1.5x at P=50, the cache works at small shapes but not at the
  production shape (HLO hash instability hypothesis — STATE #106/107 / RESEARCH Risk #5).
- Extend the scaling curve to P=50: a log(cold_jit_s) vs log(P) slope near 1.0
  indicates linear trace-time scaling (acceptable); slope ≫ 1 indicates graph
  blow-up (fix is graph-level, not cache-level).

Output:
- `cluster/21_diagnose_pscan.slurm` + `cluster/21_submit_pscan_chain.sh` committed +
  pushed.
- After cluster chain completes: `jit_scaling_sweep.csv` with up to 3 rows at
  P=5, 20, 50, all columns populated (or status=infeasible for rows where the cold
  compile timed out).
- Per (P, role) raw JAX_LOG_COMPILES logs + text-IR HLO dumps archived.
- `.planning/phases/21-benchmark-bottleneck-diagnosis/21-05-SUMMARY.md` with the
  scaling curve interpretation + cache-functioning verdict (prose only; the
  authoritative verdict is 21-07).

    data_lines = lines[1:]

    updated = False
    new_lines = [header]
    target_P = ${PROBE_P}
    for row in data_lines:

        if len(cols) != 7:
            new_lines.append(row)

        if int(cols[0]) == target_P and not cols[3]:

  ```bash
  #!/bin/bash
  # Phase 21 submit wrapper: cold + afterok-warm chain for P in {5, 20, 50}.
  # Each of the 6 jobs runs cluster/21_diagnose_pscan.slurm with appropriate
  # --export=ALL,PROBE_P,PROBE_ROLE. Wall-clock budget: each job ≤30min; 6 jobs
  # serial (warm depends on cold) → worst-case 3h across the chain (3 P values ×
  # (cold 30min + warm 30min)), but actual P=5 and P=20 should be much shorter.
  #
  # Early-stop: the user can `scancel` pending jobs if the P=5 results make
  # P=20 or P=50 unnecessary per CONTEXT.md verdict-reached-early policy.
  #
  # Usage:
  #   bash cluster/21_submit_pscan_chain.sh
  #   # (run from M3 repo root)
  #
  # Prerequisite: cluster/21_diagnose_pscan.slurm on origin/main and git pulled.
  set -u

  if [[ ! -f cluster/21_diagnose_pscan.slurm ]]; then
      echo "ERROR: cluster/21_diagnose_pscan.slurm not found. git pull origin main."
      exit 1
  fi

  declare -a P_VALUES=(5 20 50)

  declare -A COLD_JOBIDS
  declare -A WARM_JOBIDS

  for P in "${P_VALUES[@]}"; do
      echo "=============================================="
      echo "Submitting chain for P=${P}"
      echo "=============================================="

      # Submit COLD job
      COLD_OUT=$(sbatch \
          --export=ALL,PROBE_P="${P}",PROBE_ROLE=cold \
          --job-name="prl_pscan_P${P}_cold" \
          cluster/21_diagnose_pscan.slurm)
      COLD_JOBID=$(echo "${COLD_OUT}" | awk '{print $NF}')
      if [[ ! "${COLD_JOBID}" =~ ^[0-9]+$ ]]; then
          echo "ERROR: cold submit failed: ${COLD_OUT}"
          exit 2
      fi
      COLD_JOBIDS["${P}"]="${COLD_JOBID}"
      echo "Cold job submitted: P=${P} JOBID=${COLD_JOBID}"

      # Submit WARM job with afterok dependency on cold
      WARM_OUT=$(sbatch \
          --export=ALL,PROBE_P="${P}",PROBE_ROLE=warm \
          --job-name="prl_pscan_P${P}_warm" \
          --dependency=afterok:"${COLD_JOBID}" \
          cluster/21_diagnose_pscan.slurm)
      WARM_JOBID=$(echo "${WARM_OUT}" | awk '{print $NF}')
      if [[ ! "${WARM_JOBID}" =~ ^[0-9]+$ ]]; then
          echo "ERROR: warm submit failed: ${WARM_OUT}"
          exit 3
      fi
      WARM_JOBIDS["${P}"]="${WARM_JOBID}"
      echo "Warm job submitted: P=${P} JOBID=${WARM_JOBID} depends_on=${COLD_JOBID}"
  done

  echo ""
  echo "=============================================="
  echo "All 6 jobs submitted. Summary:"
  echo "=============================================="
  printf "%-6s %-12s %-12s\n" "P" "COLD_JOBID" "WARM_JOBID"
  for P in "${P_VALUES[@]}"; do
      printf "%-6s %-12s %-12s\n" \
          "${P}" "${COLD_JOBIDS[${P}]}" "${WARM_JOBIDS[${P}]}"
  done
  echo ""
  echo "Monitor with: squeue -u \$USER -n prl_pscan_P5_cold,prl_pscan_P5_warm,prl_pscan_P20_cold,..."
  echo ""
  echo "Early-stop: if P=5 results settle the verdict, cancel pending with:"
  for P in 20 50; do
      echo "  scancel ${COLD_JOBIDS[${P}]} ${WARM_JOBIDS[${P}]}"
  done
  ```
