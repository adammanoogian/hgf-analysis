<objective>
Implement phase-stratified analysis that computes effective learning rates for stable
vs volatile task phases and tests a group x phase interaction.

Purpose: ROADMAP success criterion 2 requires phase-stratified analysis showing different
effective learning rates and a testable group x phase interaction. The sim_df from batch
simulation already contains trial-level data with phase_label ("stable"/"volatile"), choice
data, and reward data. omega_2 is a session-level parameter, so phase stratification
requires a trial-level proxy for learning rate. The approach: compute the effective
learning rate as the absolute trial-to-trial change in reward probability belief (from the
HGF forward pass) and aggregate by phase. This does NOT require re-running the HGF -- it
uses the simulated belief trajectories already stored in sim_df (the beliefs field from
SimulationResult).

However, sim_df does not store per-trial beliefs directly -- it stores choices and rewards.
The simplest approach that avoids re-running HGF: compute a model-free effective learning
rate proxy as the absolute change in choice probability (using a simple Rescorla-Wagner
running estimate from the observed choices/rewards). This is the standard approach in the
computational psychiatry literature for trial-level phase analysis when the HGF is fit at
the session level.

CONTEXT.md marks phase stratification as "exploratory, not primary" and gives "Claude's
Discretion" for approach. The model-free proxy is appropriate because:
1. It avoids the complexity of re-running the HGF forward pass per participant
2. It captures the same construct (trial-level adaptation speed)
3. It is standard practice when HGF parameters are session-level

Output: New phase_stratification.py module, updated pipeline script, updated tests.

**Create src/prl_hgf/analysis/phase_stratification.py:**

This module computes a trial-level effective learning rate proxy and aggregates it by
phase_label (stable vs volatile) for each participant-session.

The effective learning rate proxy is: for each trial t, compute `alpha_t = |reward_t - p_hat_t|`
where `p_hat_t` is the running reward rate estimate for the chosen cue. Use an exponential
moving average to track `p_hat` per cue: `p_hat_{t+1,k} = p_hat_{t,k} + alpha_fixed * (reward_t - p_hat_{t,k})`
for the chosen cue k (alpha_fixed = 0.2 default; this is a smoothing parameter, not the
quantity of interest). The effective learning rate proxy per trial is the unsigned prediction
error: `|reward_t - p_hat_t|` for the chosen cue.

**Alternative simpler approach (PREFER THIS):** Since sim_df has trial-by-trial choices and
rewards, and we have phase_label per trial, compute:
- **Win-stay rate** per phase: P(repeat choice | previous choice was rewarded)
- **Lose-shift rate** per phase: P(switch choice | previous choice was not rewarded)

These are direct behavioral measures of learning rate that don't require belief
trajectory reconstruction. They are standard in the PRL literature.

Implement `compute_phase_learning_metrics`:

```python
def compute_phase_learning_metrics(
    sim_df: pd.DataFrame,
) -> pd.DataFrame:
    """Compute win-stay and lose-shift rates per participant-session-phase.

    Parameters
    ----------
    sim_df : pd.DataFrame
        Trial-level simulation data with columns: participant_id, group,
        session, trial, cue_chosen, reward, phase_label.

    Returns
    -------
    pd.DataFrame
        One row per (participant_id, group, session, phase_label) with columns:
        participant_id, group, session, phase_label, win_stay_rate,
        lose_shift_rate, n_trials.
    """
```

Implementation:
1. Sort sim_df by participant_id, session, trial
2. For each (participant_id, session) group:
   a. Compute prev_choice = choices shifted by 1 (NaN for first trial)
   b. Compute prev_reward = rewards shifted by 1
   c. win_stay: (prev_reward == 1) & (cue_chosen == prev_choice) / (prev_reward == 1)
   d. lose_shift: (prev_reward == 0) & (cue_chosen != prev_choice) / (prev_reward == 0)
3. Group by (participant_id, group, session, phase_label)
4. Aggregate win_stay_rate and lose_shift_rate as means within each group

   # --- 4b. Phase-stratified analysis ---
   print("\nPhase-stratified analysis (stable vs volatile)...")
   sim_path = _resolve_sim_path()
   if sim_path is not None and sim_path.exists():
       sim_df = pd.read_csv(sim_path)
       phase_df = build_phase_stratified_df(sim_df)

       phase_path = GROUP_ANALYSIS_DIR / "phase_stratified.csv"
       phase_df.to_csv(phase_path, index=False)
       print(f"  Saved: {phase_path}")

       # Summarise by group x phase
       summary = (
           phase_df
           .groupby(["group", "phase_label"])[["win_stay_rate", "lose_shift_rate"]]
           .agg(["mean", "std"])
       )
       print("\n  Win-Stay and Lose-Shift by Group x Phase:")
       print(summary.to_string())

       # Fit group x phase model on win_stay_rate if enough data
       n_participants_phase = phase_df["participant_id"].nunique()
       if n_participants_phase >= 6:
           import bambi as bmb  # noqa: E402 — already imported above but guard for clarity
           phase_data = phase_df.copy()
           phase_data["group"] = pd.Categorical(phase_data["group"].astype(str))
           phase_data["phase_label"] = pd.Categorical(phase_data["phase_label"].astype(str))

           for outcome in ["win_stay_rate", "lose_shift_rate"]:
               formula = f"{outcome} ~ C(group) * C(phase_label) + (1 | participant_id)"
               print(f"\n  Fitting bambi model: {formula}")
               phase_model = bmb.Model(formula, phase_data)
               phase_idata = phase_model.fit(
                   draws=args.draws, tune=1000, chains=4, random_seed=42,
               )
               phase_summary = az.summary(
                   phase_idata,
                   var_names=[v for v in phase_idata.posterior.data_vars
                              if v not in ("1|participant_id_sigma", "sigma")
                              and "1|participant_id" not in v],
               )
               print(f"\n  {outcome} model summary:")
               print(phase_summary.to_string())
       else:
           print(f"  WARNING: Only {n_participants_phase} participants. "
                 "Need >= 6 for group x phase model. Skipping.")
   else:
       print("  Simulated data not found. Skipping phase-stratified analysis.")
       print("  (Run scripts/03_simulate_participants.py first.)")
