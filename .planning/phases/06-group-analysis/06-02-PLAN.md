  2. **Create scripts/06_group_analysis.py** following the established pipeline script pattern from 03/04/05 scripts:

  - sys.path insert for project root (same pattern as other scripts).
  - Imports: config paths, load_config, group analysis functions, group_plots functions, effect_sizes functions.
  - argparse with:
    - `--model` (default "2level", choices ["2level", "3level"]) -- which model variant to analyze.
    - `--skip-plots` flag to skip figure generation (faster for CI).
    - `--draws` (default 2000) for bambi MCMC draws.
  - Main logic:
    a. Load simulated data CSV from DATA_DIR / "simulated/simulated_participants.csv".
    b. Load fitted results CSV -- look for it at RESULTS_DIR / "fit_results_{model}.csv" (or similar path from Phase 4/5 output; check the 04_fit_participants.py script to find the exact path). If not found, print clear error message and sys.exit(1).
    c. Call build_estimates_wide(fit_df, model=args.model) to pivot.
    d. Save estimates_wide to GROUP_ANALYSIS_DIR / "estimates_wide.csv".
    e. Determine params list based on model: 2level = ["omega_2", "beta", "zeta"], 3level = ["omega_2", "omega_3", "kappa", "beta", "zeta"].
    f. Determine session labels from the estimates_wide["session"].unique().
    g. Call summarize_group_models(estimates_wide, params, draws=args.draws) -- this fits bambi models for each parameter and extracts contrasts.
    h. Collect contrasts into a single DataFrame and save to GROUP_ANALYSIS_DIR / "group_contrasts.csv".
    i. Call compute_effect_sizes_table(estimates_wide, params, sessions) and save to GROUP_ANALYSIS_DIR / "effect_sizes.csv".
    j. If not --skip-plots: call plot_all_rainclouds and plot_interaction for each param, saving to GROUP_ANALYSIS_DIR / "figures/".
    k. Print summary table of contrasts and effect sizes to stdout.

  - **Phase stratification (GRP-03, exploratory):**
    l. After the main group analysis, add a section for phase-stratified analysis.
    m. The sim_df has a `phase_label` column with values "stable" or "volatile".
    n. Join estimates with phase_label: For each participant-session, compute mean omega_2 separately for stable and volatile trials. This requires going back to the trial-level sim_df and the fitted omega_2. The simplest approach: use the fitted omega_2 (which is a session-level parameter, not trial-level) and compare it across participants who had more/fewer volatile trials. But since omega_2 is session-level (same for all trials in a session), the phase stratification is better done descriptively: report how many stable vs volatile trials each session has, and note that omega_2 governs learning rate uniformly across both phases within a session. The phase effect is inherent in the HGF beliefs, not in the fitted parameter itself.
    o. Practical approach: Report this as a limitation/future direction. The HGF omega_2 is a session-level parameter, so phase stratification requires extracting trial-level belief trajectories (mu_1 changes) and comparing effective learning rates across stable vs volatile trials. This is computationally expensive (re-running the forward pass with posterior-mean params for each participant) and is marked as exploratory.
    p. Add a brief comment in the script noting this and log a message: "Phase stratification is exploratory. omega_2 is session-level; trial-level analysis deferred."

  - **test_build_estimates_wide**: Verify pivot produces correct shape (n_participants * n_sessions rows, correct columns).
  - **test_build_estimates_wide_excludes_flagged**: Verify flagged rows are dropped.
  - **test_compute_cohens_d**: With the synthetic data (group means differ by ~1.0 / sd ~0.3), Cohen's d should be > 1.0.
  - **test_compute_effect_sizes_table**: Verify output has correct columns and one row per (parameter, session).
  - **test_plot_raincloud_creates_figure**: Verify plot_raincloud returns a matplotlib Figure (don't test visual content, just that it runs).
  - **test_plot_interaction_creates_figure**: Same for interaction plot.
