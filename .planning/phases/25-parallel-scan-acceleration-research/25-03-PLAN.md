<objective>
Produce `REPRODUCTION_NOTES.md`: reproduce the published GPU benchmark for the
algorithm chosen in 25-02 BEFORE touching HGF code. Establish that:

1. We can build the algorithm correctly in JAX (numerical agreement vs the
   paper's reference within rel-err < 1e-5).
2. The convergence rate K matches the paper's published claim within +/- 1
   iteration.
3. GPU wall-clock is in the right ballpark (>2x deviation flagged as
   environmental discrepancy, not gate failure).

This is the LAST safety check before HGF prototype work in 25-04. If we cannot
reproduce a published benchmark of the chosen algorithm, we have no business
applying it to HGF.

Reproduction target (lifted from 25-02 ALGORITHM_DECISION.md Section 9):
- If PIEKS chosen: arXiv:2102.00514 Section 5 coordinated turn model at T=500,
  D=4. Reference impl: EEA-sensors/sqrt-parallel-smoothers.
- If quasi-ELK chosen: arXiv:2407.19115 Table 2 autoregressive GRU experiment.
  Reference impl: lindermanlab/elk.
- If Hybrid chosen: reproduce the underlying parallel-scan algorithm
  (PIEKS or ELK), not the hybrid sampler.

**In scope:**
- Clone the chosen reference impl into a scratch location (NOT into the project
  src/; this is exploratory).
- Set up the toy benchmark from the paper as documented (synthetic data,
  hyperparameters, hardware).
- Run reproduction on GPU via SLURM (cluster compute REQUIRED here; CPU benchmark
  would not match paper's GPU numbers).
- Record K, rel-err vs sequential reference, GPU wall-clock.
- Compare to paper's reported numbers.
- Document any environmental discrepancies (JAX version, GPU generation, etc.).
- Write reproduction notes memo with verdict.

**Explicitly NOT in scope:**
- Touching HGF code or pyhgf (that's 25-04).
- Production code under `src/prl_hgf/` (this is scratch + cluster-script work
  only).
- Optimizing the reference impl (we use it as-is; if the paper's numbers are
  with custom CUDA kernels, we accept the JAX-only deviation).
- Running the full HGF parameter recovery (that's 25-05).

job queueing, JAX-version compatibility surprises, paper details ambiguous.
A checkpoint at the end lets the user review the results before 25-04 commits
to the chosen algorithm.

before applying it to HGF. Reproduces a known-good test case so any bug in
25-04 is HGF-specific rather than algorithm-specific.
