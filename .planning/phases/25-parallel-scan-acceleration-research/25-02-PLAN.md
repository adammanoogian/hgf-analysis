Produce `ALGORITHM_DECISION.md`: a 5-way comparison of parallel-scan algorithms
for HGF likelihood evaluation, naming a single chosen algorithm and the specific
reference benchmark to reproduce in 25-03.

This plan synthesizes 25-00b (engineering feasibility) and 25-01 (theoretical
validation) into an actionable algorithm pick. PHASE_DESCRIPTION.md proposed 4
candidates; 25-RESEARCH.md §7 explicitly added PIEKS as candidate (e), citing it
as "arguably more natural for HGF than ELK." This plan must close the
PIEKS-vs-ELK question explicitly.

Candidates to compare:
- (a) ELK — Gonzalez et al. NeurIPS 2024 (arXiv:2407.19115). quasi-Newton +
  Levenberg-Marquardt + Kalman smoothing. Reference: lindermanlab/elk repo.
- (b) Vanilla DEER — Lim et al. 2023 (arXiv:2309.12252). Full Newton; simpler
  but unstable. Reference: machine-discovery/deer repo.
- (c) Zoltowski parallel-MCMC — Zoltowski, Wu, Gonzalez, Kozachkov, Linderman
  Aug 2025 (arXiv:2508.18413). Parallelizes the NUTS trajectory itself across
  sequence positions, not just within likelihood eval.
- (d) Hybrid: ELK or PIEKS for forward likelihood + sequential MCMC.
- (e) PIEKS — Yaghoobi, Corenflos, Hassan, Sarkka 2021 (arXiv:2102.00514) and
  Corenflos & Yaghoobi 2022 (arXiv:2207.00426 sq-root form).
  Reference: EEA-sensors/sqrt-parallel-smoothers repo.

Decision criteria (must be addressed for each candidate):
1. Convergence rate K (Newton iterations) for HGF given lambda_hat from 25-01.
2. Per-iteration cost (FLOPs / depth steps).
3. Covariance propagation (does the algorithm preserve Gaussian uncertainty,
   important since HGF IS a Gaussian filter under Laplace approximation).
4. API-bridging effort (cited from 25-00b feasibility verdicts).
5. Jacobian-stability tolerance (cited from 25-01 spectral-radius distribution).
6. Reference implementation availability and quality (JAX-native, published,
   tested).
7. Composability with NUTS (gradient-through-iteration support).
8. Win zone (T regime where the algorithm beats sequential).

  (2) Candidate (a): ELK
    - Paper: Gonzalez et al. NeurIPS 2024 (arXiv:2407.19115).
    - Reference impl: https://github.com/lindermanlab/elk.
    - Algorithm summary (1 paragraph): quasi-Newton + Levenberg-Marquardt trust
      region + Kalman smoothing per Newton step. Uses diagonal Jacobian
      approximation in the quasi-ELK fast path; full Jacobian only when
      diagonal fails.
    - Convergence: K iterations to 1e-5 tolerance, expected 5-10 for HGF given
      25-01 lambda_hat.
    - Per-iteration cost: O(D log T) for quasi-ELK, O(D^3 log T) for full ELK.
    - Covariance: NO (point-estimate optimizer; sigma must be tracked manually).
    - API bridging effort: cite 25-00b verdict for elk.
    - Stability: STRONG (LM trust radius prevents Jacobian-eigenvalue blow-up).
    - Reference benchmark: arXiv:2407.19115 Table 2 — autoregressive GRU,
      hidden dim 3, V100 GPU, K=10 ELK iterations, sequential 96ms vs ELK 221ms.
      NOTE: at this small D, sequential beats ELK on V100; the GPU speedup only
      manifests at larger D or larger T.
    - HGF fit: GOOD if T>>420 and lambda_hat moderately negative (per 25-01).
      MARGINAL at T=420.

  (3) Candidate (b): Vanilla DEER
    - Paper: Lim et al. 2023 (arXiv:2309.12252).
    - Reference impl: https://github.com/machine-discovery/deer.
    - Algorithm summary: full Newton's method on the recursive system; iterate
      until fixed point.
    - Convergence: K iterations, but UNSTABLE when Jacobian eigenvalues > 1
      (cite 25-RESEARCH.md §2 Direction 7 c).
    - Per-iteration cost: O(D^3 log T).
    - Covariance: NO.
    - API bridging: similar to ELK (RNN-style f(state)).
    - Stability: WEAK. 25-01 may flag failure-mode regimes where vanilla DEER
      diverges. Cite specific failure-mode parameter regimes from 25-01.
    - Reference benchmark: machine-discovery/deer demo (T=10000, 68x on V100)
      — but caveat: this is a stable (lambda < 0) example; HGF failure-mode
      regimes would need special handling.
    - HGF fit: REJECTED unless 25-01 verdict was strongly PASS with no failure
      modes; even then, ELK strictly dominates.

  (4) Candidate (c): Zoltowski parallel-MCMC
    - Paper: Zoltowski et al. Aug 2025 (arXiv:2508.18413).
    - Algorithm summary: parallelize the NUTS leapfrog trajectory + observed-
      sequence dimension simultaneously. Reported 10-30x for MALA; HMC depends
      on leapfrog depth.
    - Convergence: parallelizes the SAMPLER, not the likelihood. Composable
      with ELK/PIEKS but does not replace them.
    - Per-call cost: amortizes NUTS leapfrog across positions, but the
      likelihood eval inside each leapfrog step is still sequential unless ELK
      or PIEKS is layered underneath.
    - API bridging: HIGH effort (modifies BlackJAX NUTS implementation, not
      just the likelihood).
    - HGF fit: ORTHOGONAL to ELK/PIEKS. Considered as a follow-on phase, not
      Phase 25.
    - Recommendation: DEFER. Phase 25 prototypes the likelihood acceleration;
      sampler-level parallelization is a future phase if 25-06 verdict is
      positive.

  (5) Candidate (d): Hybrid (ELK or PIEKS forward + sequential MCMC)
    - This is the natural way to USE either ELK or PIEKS in Phase 25:
      parallel-scan replaces the likelihood eval, but the NUTS sampler stays
      sequential.
    - Effectively, candidate (d) is "candidate (a) or (e) used inside the
      production NUTS path."
    - Recommended pattern; not a separate algorithmic choice.

  (6) Candidate (e): PIEKS (sq-root form)
    - Paper: Yaghoobi et al. ICASSP 2021 (arXiv:2102.00514) +
      Corenflos & Yaghoobi 2022 (arXiv:2207.00426).
    - Reference impl: https://github.com/EEA-sensors/sqrt-parallel-smoothers.
    - Algorithm summary: iterative parallel Kalman smoother. Linearize at
      reference trajectory, run ONE parallel Kalman smoother pass (O(log T)
      via associative scan), update linearization, repeat.
    - Convergence: K iterations. For HGF (lambda_hat from 25-01), expected
      K = 3-7 (PIEKS often converges faster than ELK because the Gaussian
      filter sub-problem is better-conditioned than ELK's point-estimate
      optimization).
    - Per-iteration cost: O(D^3 log T) per Kalman smoother pass; for D=4,
      D^3=64 — small constant.
    - Covariance: YES (square-root form preserves covariances; sigma > 0
      enforced implicitly).
    - API bridging effort: cite 25-00b verdict for sqrt-parallel-smoothers.
    - Stability: STRONG (square-root form prevents covariance underflow).
    - Reference benchmark: arXiv:2102.00514 Section 5 — coordinated turn model,
      D=4 (matches HGF!), T=500, GPU benchmark showing parallel speedup.
      Also arXiv:2207.00426 sq-root version with the same benchmark.
    - HGF fit: BEST. HGF IS a Gaussian filter; PIEKS is the natural parallel
      version. Square-root numerics enforce sigma > 0 (important for HGF's
      sigma_2, sigma_3). Reference impl is JAX-native and verified.
