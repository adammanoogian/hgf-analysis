<objective>
Produce the theoretical validation deliverable `THEORETICAL_VALIDATION.md`: verify
HGF satisfies the predictability/contraction conditions of arXiv:2508.16817
(Gonzalez et al. 2025) needed for ELK/PIEKS convergence guarantees. This is the
GATE plan for the rest of Phase 25; a negative verdict triggers fallback (parallel
scan only on contracting regions, sequential elsewhere) or kills the phase.

The conditions to verify (lifted from 25-RESEARCH.md §5):

1. **Negative Largest Lyapunov Exponent (Eq. 5):**
   `lambda := lim_{T->inf} (1/T) * log(||J_T J_{T-1} ... J_1||) < 0`
2. **L-Lipschitz Jacobians (Theorem 3):**
   `||J_t(s) - J_t(s')|| <= L ||s - s'||` for all s, s' (satisfied by construction
   for HGF; sigmoid and exp are smooth — note this in the memo, no numerical scan
   needed).
3. **LLE Regularity (Eq. 10):**
   `b * exp(lambda*k) <= ||J_{t+k-1} ... J_t|| <= a * exp(lambda*k)` with a >= 1,
   b <= 1 (rules out unbounded transient growth before decay).
4. **Differentiability of f_HGF w.r.t. state (required by both PIEKS and ELK).**
   Satisfied by construction (sigmoid + exp are smooth); document and move on.

The numerical scan protocol (lifted from 25-RESEARCH.md §5):

1. Draw 100+ representative parameter vectors. PREFERRED source: Phase 14.2
   posterior chains (if available — check `.planning/phases/14*/results/` and
   `results/diagnostics/`). FALLBACK: wide prior sweep (uniform over the prior
   support set in `configs/prl_analysis.yaml`).
2. For each parameter vector, run sequential pyhgf forward on a synthetic cohort
   (T = 420, the production size) to get reference state trajectory.
3. At each trial t, compute `J_t = jax.jacobian(hgf_step)(state_t)` using the
   pure-function form lifted in 25-00b.
4. Compute `lambda_hat = (1/T) * log(||J_T J_{T-1} ... J_1||)` per parameter vector.
5. Record max singular value of J_t across trials per vector.
6. Identify failure-mode regimes: any vectors with spectral radius >= 1?

Predicted result (from 25-RESEARCH.md §1, MEDIUM confidence):
`lambda ~ -0.1 to -0.3` for typical HGF posterior support; failure modes likely at
extreme `|mu_3|` (large volatility) or extreme `omega_3` (meta-volatility tail).

    - Compute the running matrix product
      `M_T = J_T @ J_{T-1} @ ... @ J_1`. NOTE: this product can underflow or

      ~10 multiplications to keep the matrix scaled) and accumulate the log-norm
      separately. See e.g. `scipy.signal` or hand-coded:
      ```python
      log_norm_acc = 0.0
      M = J_1
      for t in range(2, T+1):
          M = J_t @ M
          if t % 10 == 0:
              q, r = jnp.linalg.qr(M)
              log_norm_acc += jnp.log(jnp.linalg.norm(jnp.diag(r))) # operator-norm proxy
              M = q  # carry over the orthogonal part only
      lambda_hat = (log_norm_acc + jnp.log(jnp.linalg.norm(M))) / T
      ```
      Alternative simpler approach: compute `lambda_hat ~= mean(log(spectral_radius(J_t)))`
      averaged over t — this is the geometric-mean spectral-radius estimator.
      Document which method is used.
    - Record max singular value across all t for this parameter vector.
    - Record num trials where spectral_radius(J_t) >= 1 (failure-mode count).
