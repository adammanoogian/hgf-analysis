  <name>Task 1: Create validation/valid03_cross_platform.py</name>
  <files>validation/valid03_cross_platform.py</files>
  <action>
  Create `validation/valid03_cross_platform.py` with the following structure:

  **Module docstring:** Explains this is VALID-03, a cross-platform consistency check. Must be run twice with different `JAX_PLATFORM_NAME` env vars and results compared.

  **1. `run_fit_and_save(output_path: Path, n_per_group: int = 3, n_chains: int = 2, n_draws: int = 200, n_tune: int = 200, seed: int = 42) -> None`:**
  - Import `jax` and print `jax.devices()` to log which platform is active.
  - Build a small cohort config: use `make_power_config(load_config(), n_per_group, effect_size_delta=0.5, master_seed=seed)` to get a frozen config.
  - Simulate: `sim_df = simulate_batch(cfg)`.
  - Fit 3-level model only (sufficient for cross-platform check): `idata = fit_batch_hierarchical(sim_df, "hgf_3level", n_chains=n_chains, n_draws=n_draws, n_tune=n_tune, target_accept=0.9, random_seed=seed, sampler="numpyro", progressbar=False)`.
  - Extract per-participant posterior means for each parameter.
    - Use `idata.posterior.coords["participant"].values` for IDs.
    - For each param in `idata.posterior.data_vars`: compute `float(idata.posterior[param].mean(dim=["chain", "draw"]).isel(participant=i))` for each participant i.
  - Build dict: `{"platform": str(jax.devices()[0].platform), "n_per_group": n_per_group, "seed": seed, "n_chains": n_chains, "n_draws": n_draws, "posterior_means": {param: {pid: mean_val, ...}, ...}}`.
  - Save to `output_path` as JSON with `json.dump`.
  - Print summary: platform, n_participants, param names.

  **2. `compare_results(path_a: Path, path_b: Path, rtol: float = 0.01) -> bool`:**
  - Load both JSONs.
  - For each parameter, for each participant, compute relative error: `abs(mean_a - mean_b) / (abs(mean_a) + 1e-8)`.
  - Collect all relative errors. Print summary (max, mean, per-param breakdown).
  - Return True if ALL relative errors < `rtol`, else False.
  - Print clear PASS/FAIL message.

  **3. CLI `if __name__ == "__main__":`:**
  - `argparse` with subcommands:
    - `run`: calls `run_fit_and_save(args.output)` with optional `--n-per-group`, `--seed`, `--n-chains`, `--n-draws`, `--n-tune`.
    - `compare`: calls `compare_results(args.path_a, args.path_b, args.rtol)` with `--rtol` defaulting to 0.01.
  - Exit code 0 on pass, 1 on fail.

  **Usage pattern (documented in docstring):**
  ```bash
  # Run on CPU:
  JAX_PLATFORM_NAME=cpu python validation/valid03_cross_platform.py run --output results/valid03_cpu.json
  # Run on GPU (default on GPU node):
  python validation/valid03_cross_platform.py run --output results/valid03_gpu.json
  # Compare:
  python validation/valid03_cross_platform.py compare results/valid03_cpu.json results/valid03_gpu.json
  ```

  **Imports:** `from __future__ import annotations`, `argparse`, `json`, `sys`, `pathlib.Path`, `time`. Deferred imports for heavy libs (jax, prl_hgf) inside the functions. Add `sys.path.insert(0, str(Path(__file__).resolve().parent.parent))` for project root access.

  **Style:** NumPy-style docstrings, type hints, line length 88.
